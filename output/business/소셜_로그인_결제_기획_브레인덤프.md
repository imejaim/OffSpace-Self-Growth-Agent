## 1. 로그인 시스템

### 구글 로그인
- 간편하게 구글 로그인으로 접속 가능하도록

### 비로그인 끼어들기
- 로그인 없이도 끼어들기 가능
- 닉네임 직접 입력 가능: '신입사원', '윤사원' 등
- 닉네임 미입력 시 랜덤 부여: '지나가던 나그네', '아무개', '옆집아저씨' 등 풀에서 랜덤

### 캐릭터 반응 차이 (핵심 재미 요소)
- **비로그인 유저 끼어들기 시:**
  - 코부장: "뉘신지?" 하면서도 답변
  - 오과장: "아무튼..." 하면서 답변
  - 모르는 사람 취급하면서도 친절하게 대응
- **로그인 유저 끼어들기 시:**
  - 아는 사이로 인식
  - "초천재 회장님!" "대표님, 언제 오셨지?" 등 친근한 반응
  - 유저별 호칭/관계 기억

## 2. 결제 시스템

### 현재 상태 (확인 필요)
- PayPal 연동 시작됨 (docs/payment/PAYPAL_SETUP.md 참조)
- 쿠폰 발행도 진행했었음 (확인 필요)
- 간편결제, 간편계좌이체 검토했었음

### 결제 수단
- **PayPal**: 해외 결제 (이미 셋업 시작)
- **간편결제**: 카카오페이, 네이버페이 등 (토스페이먼츠 없이도 가능)
- **간편계좌이체**: 직접 계좌이체 (토스페이먼츠 비용 회피)
- 토스페이먼츠는 비용이 비싸서 제외

### 과금 모델 구상
- **끼어들기 종량제**: 10회 끼어들기 = $1 (약 1,300원)
  - 첫 끼어들기는 무료로 답변
  - 이어서 파고들기(추가 끼어들기) 시 과금
- **월 구독 모델**: 하루 N회 끼어들기 포함
  - API 비용 대비 구독 가격 산정 필요
  - 사업성 분석 필수

### 비용 분석 과제
- 끼어들기 1회당 API 비용 산정
- 10회/$1 모델의 마진 계산
- 월 구독 시 일일 끼어들기 한도와 월 구독 비용 산정
- 저장 비용 (대화 히스토리 보관)

## 3. 소셜 기능 — 2라인 구조

### 핵심 개념: 개인 라인 + 공개 라인

#### 개인 라인 (My Line)
- 오늘의 소식지를 보고 끼어들기
- 나만의 대화, 파고들기 내용
- 내일도 다시 볼 수 있어야 함 → 저장 기능
- 검색 기능 (과거 끼어들기 검색)
- "저장하기" = 나만 볼 수 있는 프라이빗 저장 (구독자 전용 or 별도 과금)

#### 공개 라인 (Social Line)
- "공개하기" 버튼 → 내 끼어들기 내용이 공개 라인으로 이동
- **Pretext 효과**: 글자가 복사되어 날아가는 애니메이션으로 공개 라인에 착지
- 좌우 스와이프로 개인 라인 ↔ 공개 라인 전환 (앱/웹 UX)
- 공개된 끼어들기에 다른 사람이 추가 끼어들기 가능

#### 기본 공개 정책
- **기본값은 공개** (소셜 활성화 우선)
- 나만 보고 싶으면 "Private 저장" 선택
- 이래야 소셜 피드에 콘텐츠가 쌓임

#### 팔로우 시스템
- 깊이 있게 파고드는 사용자를 팔로우 가능
- AI와 함께하는 개인 소식지이면서 소셜 뉴스

### 댓글 시스템 문제 회피 (이전 논의)
- 단순 댓글 형태가 되면 비방/어뷰징 발생
- "끼어들기"는 AI 캐릭터가 중재하므로 자연스러운 톤 유지
- 사람 간 직접 대댓글이 아닌, AI 대화에 참여하는 구조
- → 기존 댓글 시스템의 독성 문제를 구조적으로 회피

## 4. 소식지 구조 개편

### 3개 섹션 (변경)
1. **핫뉴스** — 주요 뉴스, 공식 소스 중심
2. **선택주제** — 사용자가 직접 주제를 바꿀 수 있음 (아래 상세)
3. **뒷담화** — 젬대리 담당, 비공식 소스 중심

### 뒷담화 섹션 (젬대리 캐릭터)
- **소스**: Reddit, 블라인드, 카페, X.com, 인스타, 유튜브, 개인 블로그
- **톤**: 빠른 반응, 리얼한 이야기, 공식적이지 않은 정보
- **레퍼런스 처리**:
  - 가능하면 링크 첨부
  - 저작권 이슈 시 "여기서 이걸로 검색하면 나와요" 식으로 안내
  - 캡처 살짝 첨부 가능

### 주제 자유 교체 (핵심 기능)
- 카테고리 제목을 클릭하면 수정 가능
- "1. AI 핫뉴스" → 사용자가 원하는 주제로 변경
- 안내 문구: "원하는 주제를 적어보세요. 연예인, 경제, 등등..."
- 랜덤 주제도 가능
- **비정상 입력 대응** (예: 'ㅋㅋㅋ'):
  - 코부장: "이 주제는 뭐지? 오과장 뭐 조사한 거 있어?"
  - 오과장: "ㅋㅋㅋ의 어원에 대해서 조사한 바에 따르면..."
  - 젬대리: "ㅋㅋㅋ 뭐 저 오늘 재밌는 이야기 들었는데요..."
- → 어떤 입력이든 캐릭터답게 대응하는 것이 핵심
- 주제도 끼어들기처럼 자유롭게 바꿀 수 있어야 진정한 끼어들기

### 발행 시스템
- "발행하기" 버튼 → 해당 주제에 대해 소식지 생성
- 접속 시 "오늘의 소식 발행" 버튼으로 트리거
- 구독자는 여러 번 다른 주제로 발행 가능
- 3개 이상 주제 추가도 가능 (구독 등급에 따라)
- UI/UX 사용 순서 설계 필요

## 5. 비전 요약

> "진정한 개인형 AI 소식지이면서, 함께하는 AI 소셜 소식지"

- 지금: 대표님 전용 티타임 AI 소식지
- Phase 1: 로그인 + 결제 → 다른 사용자도 사용 가능
- Phase 2: 저장 + 검색 → 개인화된 소식지 경험
- Phase 3: 공개 + 팔로우 → 소셜 뉴스 플랫폼
- 최종: AI와 함께하는 참여형 소셜 뉴스
