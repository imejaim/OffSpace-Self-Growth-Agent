# Offspace 티타임 발행 규칙

> 티타임/야식타임 콘텐츠 작성 시 반드시 이 문서를 먼저 읽고 따를 것.
> 최종 업데이트: 2026-04-06

---

## 1. 고정 카테고리 구조

카테고리는 **고정**이다. 매 발행마다 동일한 카테고리를 유지하고, 뒤에 흥미를 끄는 서브 타이틀을 붙인다.

```
## N. [고정 카테고리] — "서브 타이틀"
```

| 순번 | 고정 카테고리 | 다루는 내용 | 절대 안 섞는 것 |
|------|-------------|------------|---------------|
| 1 | **AI 핫뉴스** | 그 주의 가장 큰 뉴스, 빅테크 동향, 투자/인수/IPO, 경영진 변동 | — |
| 2 | **AI 에이전트** | 에이전트 프레임워크, 표준(MCP, UCP 등), 배포 사례, 에이전트 보안 | IPO, 경영진 인사 등 기업 비즈니스 뉴스 |
| 3 | **AI 논문과 모델** | 새 모델 릴리즈, 벤치마크, 오픈소스, 추론 인프라(vLLM 등) | 순수 비즈니스/투자 뉴스 |
| 4 | **AI 로봇 / 피지컬 AI** | 휴머노이드, 자율주행, 온디바이스 하드웨어, 로봇 공학 | 순수 소프트웨어 뉴스 |
| 5 | **보너스** 또는 **그 외** | 규제, 사회 이슈, 여론, 재미 소식, 오프스페이스 관련 | — |

**카테고리 분류 원칙**: 뉴스가 어느 카테고리에 속하는지 헷갈리면, **그 뉴스의 핵심이 무엇인지** 기준으로 판단한다.
- OpenAI COO 인사이동 → "AI 핫뉴스" (기업 경영 뉴스)
- MCP 표준 업데이트 → "AI 에이전트" (에이전트 프로토콜)
- 새 모델 벤치마크 → "AI 논문과 모델"

**야식타임**도 동일한 카테고리 체계를 따른다. 이모지를 카테고리 앞에 붙이는 것은 자유.

## 2. 출처 다양성 규칙

### 수량 기준

| 항목 | 최소 기준 |
|------|----------|
| 토픽당 참고 링크 | **3개 이상** |
| 전체 티타임 총 출처 | **12개 이상** |
| SNS/커뮤니티 출처 | 전체 중 **2~3개 이상** |

### SNS/커뮤니티 소스 필수

공식 뉴스만 다루지 않는다. **발빠른 소식은 커뮤니티에서 먼저 나온다.**

필수 커버 채널:
- **Reddit** (r/MachineLearning, r/LocalLLaMA, r/artificial 등)
- **X.com** (AI 연구자, 기업 공식 계정)
- **YouTube** (AI 채널, 데모 영상)
- **HackerNews** (ycombinator)
- **GitHub** (릴리즈, 이슈, 트렌딩)
- **Discord** (AI 커뮤니티)

### 출처 표기 형식

```markdown
> 📎 참고 링크
> - [매체명 날짜](URL) · 신뢰도 ★~★★★★★
> - [Reddit r/LocalLLaMA 4/5](URL) · ★★★
> - [YouTube @channel 4/4](URL) · ★★★
```

### 시간 표기

**(발생일 · 보도일)** 병기 필수. 커뮤니티에서 먼저 퍼진 경우 커뮤니티 최초 발견일도 표기.

예: `(3/31 발생 · 4/1 Reddit · 4/4 공식 보도)`

## 3. 이미지 규칙

| 항목 | 기준 |
|------|------|
| 전체 티타임 이미지 수 | **최소 2장** |
| 배치 위치 | 관련 토픽 시작부 (제목 바로 아래) |
| 허용 소스 | 뉴스 기사 대표 이미지, 공식 블로그, 제품 스크린샷, 데모 캡처 |
| 금지 | AI 생성 일러스트 (너무 깔끔하면 안 됨, 아마추어 감성 유지) |
| 모바일 | 이미지 위 + 대화 아래 |
| 웹 | 좌우 배치 고려 (NYT 레이아웃 참고) |

## 4. 캐릭터별 소스 채널 분담

| 캐릭터 | 주 채널 | 역할 |
|--------|---------|------|
| **젬대리** | Reddit, YouTube, X.com, GitHub, Discord | 커뮤니티에서 발빠르게 캐치, 흥분하며 공유. "아 이거 레딧에서 봤어요!" |
| **오과장** | HackerNews, Crunchbase, 시장 보고서, Deloitte/Gartner | 팩트와 숫자 보강, 기획적 시각 |
| **코부장** | 공식 블로그, 논문(arXiv), 기술 문서, TechCrunch | 기술적 분석, 마무리 정리 |

**대화 흐름**: 젬대리가 SNS에서 먼저 캐치 → 오과장이 팩트/숫자 보강 → 코부장이 기술적 분석/마무리

**금지**: 한 캐릭터가 혼자 독백으로 설명하는 패턴. 끼어들기 답변도 최소 2명 이상 반응.

## 6. 품질 체크리스트 (발행 전 확인)

- [ ] 고정 카테고리 5개 유지했는가?
- [ ] 각 카테고리에 해당 주제만 담겼는가? (AI 에이전트에 IPO 뉴스 넣지 않았는가?)
- [ ] 이미지 최소 2장 포함했는가?
- [ ] 토픽당 출처 3개 이상인가?
- [ ] 전체 출처 12개 이상인가?
- [ ] SNS/커뮤니티 출처 2~3개 이상 포함했는가?
- [ ] 젬대리가 커뮤니티 소식을 전달하고 있는가?
- [ ] (발생일 · 보도일) 병기했는가?
- [ ] 한 캐릭터 독백 없이 대화 형식인가?
- [ ] 끼어들기 답변은 최소 2명 이상 반응하는가?

## 7. Vol.2 (4/3)가 좋았던 이유 — 레퍼런스

- 토픽당 링크 3개 이상, 총 15개 출처
- 이미지 2장 (NVIDIA 로봇, 휴머노이드)
- 끼어들기에 팀 전체가 반응 (독백 없음)
- 카테고리별 소식이 정확히 분류됨
- 표, 비교 데이터 등 시각적 요소 풍부

이 수준을 매 발행의 **최소 기준**으로 삼는다.
